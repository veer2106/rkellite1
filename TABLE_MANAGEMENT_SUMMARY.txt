🚀 API ENDPOINTS
═══════════════════════════════════════════════════════════

1. GET /api/tables
   Get all tables with optional filters
   Query: ?section=lodge-dine&status=available

2. GET /api/tables/config
   Get table counts for both sections
   Returns: { "lodge-dine": 10, "cafe-restaurant": 15 }

3. GET /api/tables/availability/:section/:tableNumber
   Check if specific table is available
   Example: /api/tables/availability/lodge-dine/5

4. POST /api/tables/setup (Admin)
   Setup tables for a section
   Body: { section, numberOfTables, seatsPerTable }

5. PUT /api/tables/:id/status
   Update table status
   Body: { status: "cleaning" }

6. DELETE /api/tables/:id (Admin)
   Delete a table (if no active order)

7. POST /api/orders (Updated)
   Create order with table validation
   Checks table availability for dine-in orders

8. PUT /api/orders/:id/status (Updated)
   Update order status
   Frees table when status = completed/cancelled

  Waiter → POS → Dine-In
  Section: Lodge-Dine
  Table: 5 (now available!)
